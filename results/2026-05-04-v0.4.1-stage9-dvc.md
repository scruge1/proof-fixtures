## Why Hetzner Object Storage and not HuggingFace private datasets

| Surface | Data residency | License/compliance | Decision |
|---|---|---|---|
| Hetzner Object Storage Nuremberg | DE (intra-EEA) | Same vendor as compute, no new sub-processor entry needed | **PICK** |
| HF private datasets | AWS US-East | Schrems II / SCCs Module 3 / DPF tail; D26-PIVOT removed paid-API stack precisely to avoid this | rejected |
| AWS S3 (eu-west-1) | IE | Schrems II OK in EU but new sub-processor entry; no compute synergy | rejected |
| Backblaze B2 | US/EU | New sub-processor; compliance overhead | rejected |

## License posture

- DVC: Apache-2.0
- s3fs: BSD-3
- aiobotocore + botocore: Apache-2.0 (community fork stays in line with AWS SDK)
- Hetzner Object Storage: Hetzner is already DPA v0.4 §7.1 sub-processor for compute; Object Storage is the same vendor's storage surface, no new sub-processor entry needed.
